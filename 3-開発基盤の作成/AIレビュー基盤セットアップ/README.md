AI レビュー基盤セットアップ
=========================

目的
----
AI を用いたコードレビュー・品質チェックの仕組みを整備するための設計メモ。現在は GitHub 単体では完全自動化が難しいため、ローカルで差分取得スクリプトを実行し、その結果を AI に渡す運用を前提とする。

レビュー種別と入力
------------------
- 実装レビュー: 変更前状態と、コミット済みの最新 HEAD との差分 diff を取得し、機能作成指示書（`function.md`）などのコンテキストとともに AI に渡す。
- テストレビュー: （後述）テスト観点や不足ケースをチェックするワークフローを別途定義する。
- メタレビュー: 定期的な振り返りを行う計画。具体的なメソッドは未定のため後続で整理する。

記載すべき内容
--------------
- 役割と範囲: AI がレビューする対象（実装/テスト/設計妥当性など）と除外範囲。
- 入力コンテキスト設計: 差分、関連ファイル、`function.md` 等の渡し方と要約方針。
- プロンプト設計: 指摘優先度、トーン、出力フォーマット、禁止事項、例示。
- ワークフロー: 実行タイミング（PR 時・手動・定期）、ローカルスクリプトや CI との連携方法。
- フィードバック形式: コメント粒度、ブロッカー/警告/参考のレベル分け、修正提案の扱い。
- セキュリティ・プライバシー: 秘匿情報を含めないためのフィルタリング方針、ログ保全ポリシー。
- 評価と改善: レビュー精度の測定指標、サンプルセット、改善サイクル（プロンプト/モデル更新手順）。
- ツール連携: GitHub/GitLab のボット設定、チャットツール通知、ローカル CLI など運用手段。

運用メモ
--------
- 差分取得スクリプトと入力テンプレートを整備し、誰でも同じ手順でレビューを回せるようにする。
- 初期導入時はスコープを限定し、誤検出が多いチェックは緩和または手動実行にする。
- 定期的にレビュー結果をサンプリングし、プロンプトや入力コンテキストを改善する。
