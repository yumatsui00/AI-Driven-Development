CI セットアップガイド
===================

目的
----
プロジェクトの品質チェックを自動化するための CI/CD 設計メモをまとめる。再利用できるワークフロー、チェックリスト、設定例を蓄積する。

記載すべき内容
--------------
- 対象リポジトリ/ブランチ戦略: 保護ブランチ、必須ステータスチェック。
- ワークフロー構成: 実行タイミング（push/pr/schedule）、ジョブ分割方針、キャッシュ戦略。
- チェック項目: Lint/Formatter、ユニット・統合テスト、型チェック、セキュリティ/ライセンススキャン。
- 環境設定: Node/Java/Python などのバージョン固定、依存解決の手順、秘密情報の扱い（Secrets/環境変数）。
- 成果物: ビルドアーティファクト、テストレポート、カバレッジの保存と公開方法。
- フィードバック: PR コメント、ステータスバッジ、失敗時のログ収集方法。
- テンプレート: 汎用的なワークフロー YAML、Make/Turbo など補助スクリプトの例。

運用メモ
--------
- 新しいチェックを追加するときは、実行時間と失敗時のデバッグ手順を必ず記載する。
- フレーク対策（リトライ/タイムアウト/並列度）やキャッシュ無効化手順を残し、トラブル時に素早く切り替えられるようにする。
